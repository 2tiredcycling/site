# V4 范围说明

适用版本：`V4`

更新时间：2026-03-21

## 1. 版本目标

V4 的目标是将当前 GPX 路线平台升级为“社团官网 + 路线中心”统一入口，形成可持续运营、可扩展迭代、可稳定维护的站点基础。

## 2. 范围边界

### 2.1 In Scope（V4 必做）
- 官网主导航：`首页 / 社团介绍 / 管理团队 / 活动 / 路线 / 反馈 / 联系`
- GPX 路线中心继续作为官网核心功能模块
- 主题化前端与官网样式统一（含页面风格、色系、视觉资源）
- 活动展示与活动详情能力持续完善
- 后台可维护基础内容（公告、页面文案、首页展示数据）

### 2.2 Out of Scope（V4 暂不做）
- 复杂 CMS（可视化拖拽、多语言完整体系、版本树）
- 自动推荐、A/B 实验等高阶运营系统
- 大规模商业化会员体系与支付闭环

## 3. 模块边界：官网与 GPX

### 3.1 官网域
- 负责社团信息展示、活动入口、联系反馈、加入引导

### 3.2 GPX 域
- 负责路线浏览、路线详情、下载、反馈与活动路线关联展示

### 3.3 边界原则
- 官网可引用 GPX 数据，但不侵入 GPX 核心业务逻辑
- GPX 核心数据结构稳定优先，官网能力通过扩展实现

## 4. 阶段拆分（按当前共识）

- `V4.0（计划阶段）`
  - 完成范围冻结、信息架构、数据与权限设计
  - 明确文档基线与迭代边界

- `V4.1（框架阶段）`
  - 搭建官网基础框架与核心路由
  - 完成首页、社团介绍、管理团队、联系等基本页面骨架

- `V4.2（美观阶段）`
  - 完成整体视觉升级与样式统一
  - 完成主题配色体系、首页与子页面风格一致化
  - 完成核心页面的交互细节优化

- `V4.3（新界面阶段，计划中）`
  - 新增“加入社团申请界面”
  - 新增“未来活动界面”
  - 新增“活动报名界面”
  - 与现有活动/路线/联系流程打通

## 5. 当前版本定位

- 已完成重点：`V4.0 ~ V4.2`
- 下一阶段重点：`V4.3` 新界面与业务流程补全

## 6. 约束与假设

- 技术栈沿用 Flask + SQLAlchemy + Alembic
- 继续兼容 SQLite（开发）与 PostgreSQL（生产）
- 所有新增前台页面默认支持移动端访问
- 发布采用“先主干验证、再 tag 发布”流程
