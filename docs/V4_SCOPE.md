# V4.0 范围说明

适用版本：`V4.0`

更新时间：2026-03-19

## 1. 版本目标

V4.0 将当前 GPX 路线平台升级为“社团官网 + 路线共享中心”的统一站点，确保文化节前可对外展示、可持续运营、可安全管理。

## 2. 范围边界

### 2.1 In Scope（V4 必做）
- 官网主导航：`首页 / 社团介绍 / 管理团队 / 活动 / 路线 / 反馈 / 联系`
- GPX 路线中心继续作为官网子模块保留
- 后台新增内容管理能力：公告、静态页面内容、首页配置
- 活动报名基础流程：前台提交、成功页、后台可查看
- 权限模型覆盖新模块（页面级 + 操作级）
- SEO 基线：`title`、`description`、`sitemap.xml`、`robots.txt`、404
- 可重复迁移脚本与基础测试补齐

### 2.2 Out of Scope（V4 不做）
- 复杂 CMS（多语言、可视化拖拽、版本树）
- 高级运营能力（自动推荐、A/B、复杂工作流）
- 非核心增强项（媒体库高级功能、定时发布、高级告警等）放入 V4.4+

## 3. 模块边界：官网与 GPX

### 3.1 官网域
- 负责品牌展示、社团信息、对外联系、活动展示与报名入口

### 3.2 GPX 域
- 负责路线查询、路线详情、下载、反馈、活动与路线关联展示

### 3.3 边界原则
- 官网页面可引用 GPX 数据，但不直接承载 GPX 核心业务逻辑
- GPX 模块数据结构稳定优先，官网能力通过新增表扩展，不侵入既有路线核心表

## 4. 阶段拆分

- `V4.0`：范围冻结 + IA + 数据模型与迁移基线 + 设计输入
- `V4.1`：官网前台 MVP（页面上线）
- `V4.2`：后台内容管理 + 活动报名闭环
- `V4.3`：发布与验收收官（测试、安全、运维、文档）
- `V4.4+`：稳定增强项

## 5. 文化节前必须完成（P0 映射）

对应 `docs/V4_0_TASKLIST.md` 与后续 minor 计划，文化节前至少完成：
- 前台核心页面可访问且可维护
- 活动报名流程可用
- 权限控制有效
- 安全基线保持
- 基础 SEO/部署/监控可用

## 6. 约束与假设

- 继续沿用 Flask + SQLAlchemy + Alembic 技术栈
- 继续兼容 SQLite（开发）与 PostgreSQL（生产）
- 所有新页面默认移动端可用
