# V4.0 后台内容管理模块设计

适用版本：`V4.0`

更新时间：2026-03-19

## 1. 模块目标

让非开发管理员可以在后台维护官网内容，减少“改文案必须改代码”的依赖。

## 2. 后台菜单结构

- 管理总览 `/manage`
- 内容管理（新）
  - 公告管理 `/manage/announcements`
  - 静态页管理 `/manage/pages`
  - 首页配置 `/manage/homepage`

## 3. 页面结构

## 3.1 公告管理
- 列表：标题、状态、置顶、发布时间、更新时间、操作
- 编辑：标题、正文、状态、置顶、排序、发布时间
- 操作：保存草稿、发布、下线、删除

## 3.2 静态页管理
- 列表：slug、标题、状态、更新时间
- 编辑：标题、摘要、正文（支持基础 Markdown 或纯文本）
- 操作：保存、发布、下线

## 3.3 首页配置
- 配置项：Hero 文案、快捷入口开关、公告区开关、模块排序
- 存储：`homepage_sections` 的 `payload_json`
- 操作：预览（后续）、保存

## 4. 内容发布流程

1. 编辑为 `draft`
2. 审核/确认后切换 `published`
3. 临时撤下切换 `offline`

## 5. 回滚策略

- V4.0 不做复杂版本树
- 依赖审计日志记录字段变更
- 重大内容修改建议先导出 JSON 备份

## 6. 与现有 manage 风格一致性

- 保持统一头部版本号显示
- 复用现有 `card`、`table`、`form-card` 样式体系
- 沿用 CSRF 与 flash 提示机制
