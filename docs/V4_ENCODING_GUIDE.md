# V4.0 编码与中文文案规范

适用版本：`V4.0`  
更新时间：2026-03-19

## 1. 目标

彻底消除 `????`、乱码与编码不一致导致的显示异常。

## 2. 统一编码要求

- 代码文件：UTF-8（无 BOM）
- HTML 模板：必须包含 `<meta charset="UTF-8">`
- CSV 导出：若面向 Excel，可使用 UTF-8 BOM；其余默认 UTF-8
- 数据库存储：统一 UTF-8 编码字符集

## 3. 开发规范

- 禁止在源码中保存“被错误转码后的中文”
- PR 自检：
  - 页面标题
  - Flash 文案
  - 角色标签
  - 审计日志中文内容

## 4. 数据库与运行时检查

- SQLite：确认文本字段按 UTF-8 正常读取
- PostgreSQL：确认数据库与连接参数使用 UTF-8
- 日志链路：确保终端、容器、文件采集均为 UTF-8

## 5. 乱码排查 checklist

1. 浏览器页源码检查 charset
2. Python 源文件编码检查
3. 数据库中原始值抽样核对
4. 反向代理/容器日志编码核对
5. 导入导出文件编码核对

## 6. V4.0 验收标准

- 管理后台中文文案无乱码
- 公开页面中文文案无乱码
- 关键日志（登录失败、安全告警）中文可读
- 新增文档全部 UTF-8 保存
