# V4.0 测试计划（为 V4.1/V4.2 实施准备）

适用版本：`V4.0`  
更新时间：2026-03-19

## 1. 测试目标

在 V4.1/V4.2 开发前明确测试边界，确保新增官网能力不会破坏现有 V3 稳定性。

## 2. 范围

## 2.1 回归范围（必须）
- 登录与会话
- 权限矩阵与页面守卫
- 活动列表/详情
- 路线查询与下载
- 安全拦截与限流基础行为

## 2.2 V4 新增范围（计划）
- 静态页内容发布
- 公告发布与展示
- 首页配置读取
- 活动报名提交与后台状态流

## 3. 测试层级

- 单元测试：字段校验、状态机、权限判定函数
- 集成测试：路由、模板渲染、数据库写入、权限拦截
- 冒烟测试：首页、活动、路线、后台登录关键链路

## 4. 用例清单（最小集）

1. 未登录访问 `/manage/*` 被拦截  
2. 无权限用户访问新增后台页面返回 `403`  
3. 公告从 `draft` 发布后前台可见  
4. 静态页发布后前台按 slug 可见  
5. 活动报名提交成功写入 `event_registrations`  
6. 报名频控生效并提示重试  
7. 管理员可将报名状态改为 `confirmed/rejected`  
8. `/download/<id>` 继续可用并统计下载数  

## 5. 覆盖率建议

- 关键权限分支覆盖率 >= 90%
- 新增数据表核心 CRUD 分支覆盖率 >= 80%

## 6. 执行节奏

- V4.1 开发期间：每个 PR 跑现有 `pytest -q`
- V4.2 报名功能接入时：新增集成测试并纳入 CI
- V4.3 收官前：执行一次全量回归 + 安全冒烟
